# 📞 콜팀(상담팀) — 담당 구간 업무 상세

> **담당 Phase**: 유입(P1) → 조회/신청(P2) → 선별/배정(P3) → 상담/TM(P4)  
> **핵심 목표**: 광고/소개 유입 고객을 미팅 인계까지 완전 전환  
> **팀 구성**: 
> - **콜팀원** (call_member): TM 실행, 고객 상담, 스크립트 진행, 일정 예약  
> - **콜팀장** (call_lead): DB 배정 관리, 품질 모니터링, 일과 보고

| 체크 항목 | 확인 내용 | 시스템 기록 |

| ✓ 심장질환 이력 확인 | "심장질환(심근경색, 협심증 등) 있으신가요?" | 심장 여부 선택<br/>있으면 → 자동 진행불가 |
| ✓ 보험 상태 확인 | "월 7만원 이상 보험료 내고 계신 상품 있으신가요?" | 보험료 금액 입력<br/>미납/실효 여부 확인<br/>계약자=납입자 확인 |

| ✓ 수수료 안내 | "저희는 환급금의 10% 수수료를 안내드리고 있습니다" | 수수료 10% 안내 완료 체크 |

## 소개 여정 (R1~R3) — 콜팀 담당 구간 **차이점**

### 개요
- **R1 (소개유입)** ~ **R3 (환급신청)**: S1~S3과 동일 프로세스
- **R4 (사전분석)**: **선별/TM 스킵** → Same-owner 자동배정으로 바로 진행
- 콜팀장 배정, 콜팀원 TM 작업 **생략**

### 콜팀 작업 생략 이유
- 추천인이 기존 고객 → 신뢰도 높음
- 동일 영업사원(Same-owner)에게 자동 배정
- TM 전환율 불필요 (이미 신뢰 구축됨)

### 콜팀 작업 (있는 부분)
- S1~S3: 유입/조회/신청 모니터링 (동일)
- S4 선별/배정: **없음** (자동배정)
- S5~S6 TM: **없음** (스킵)

## 간편청구 여정 (Q1~Q2) — 콜팀 담당 구간 **최소화**

### 개요
- **Q1 (접수시작)** ~ **Q2 (초기분기/본인확인)**: 콜팀 관여 최소
- 청구팀(claims_member)이 주 담당
- 콜팀은 **필요시 보조 통화** 만 지원

### 콜팀 작업
- Q1~Q2: 청구 고객 본인확인 통화 (선택사항)
- 나머지 Q3~Q9: 청구팀 전담

## 주요 시스템 인터페이스

### 1. 신청 접수 (Requests.tsx)
- S1~S3 신청 현황 조회
- 선별 대상 DB 리스트 표시

### 2. 1차 TM (FirstTM.tsx)
- 배정받은 고객 목록
- 상담 스크립트 체크리스트
- 고객정보, 심평원 이력 조회

### 3. 상담 (Consultation.tsx)
- 1차 TM 스크립트 입력 폼
- 심평원 이력 첨부
- 동반신청 고객 입력
- 녹취 파일 업로드
- 콜 결과 기록

### 4. 2차 TM (SecondTM.tsx)
- 2차 상담 고객 목록
- 미팅 일정 확정 입력
- 미팅 유형 (대면/비대면) 선택

### 5. 인계 (Handoff.tsx)
- 인계 폼 작성
- 특이사항 기록
- 최종 체크리스트

### 6. TM 체크리스트 (TMChecklist.tsx)
- 콜팀원별 처리 현황
- 인계 서류 검증
- 품질 점검

### 7. 일일 보고 (DailyReport)
- 배정 현황 요약
- 부재 건 관리
- 콜팀원 일과 성과

## 참고: 미구현 항목 및 개선안

### 현재 시스템의 Gap

1. **DB 분류 프로세스 미구현**
   - 가능DB/보상DB/소개DB 자동 분류 로직 필요
   - 수동 선택 UI 추가 필요

2. **정기 배정 자동화 미구현**
   - 현재: 단건 배정만 가능
   - 필요: 대량 배정 자동화 (10:00/13:00/15:00)

3. **암/뇌/심 자동 진행불가 처리 미구현**
   - 현재: 콜팀원이 수동으로 판단
   - 필요: 심평원 조회 결과 자동 매칭 및 진행불가 자동 처리

4. **Cut-rule 기준 미정의**
   - 어떤 고객을 Cut할지 기준 명확화 필요

5. **배정/모니터링 대시보드 미구현**
   - 콜팀장 실시간 현황 모니터링 대시보드 필요
   - 콜팀원별 처리율, 거절률, 재배정률 시각화

6. **조회 실패/이탈 추적 미구현**
   - S2 심평원 조회 실패 현황
   - S2→S3 신청 이탈률
