# ⚙️ 관리/지원팀 — 전 여정 횡단 업무

> **역할**: 직접 고객 대응은 없으나 전 여정에서 운영 지원 및 준법 감독  
> **핵심 키워드**: 위촉관리 / 계약보전 / 준법감시 / 개인정보보호  
> **정기 활동**: 월 1회 교육, 월 1회 현장감사, 주간 모니터링

---

## 1. 총무/인사팀 업무 (Admin)

### 1.1 위촉/해촉 관리

**정기 업무**
- 신규 설계사 협회 등록 절차 지원 (서류 검수, 제출)
- 보험설계사 협회 조회 (보험대리점협회, 손보협회 등)
- 위촉장 발급 및 보관
- 해촉 요청 접수 및 처리
- 설계사 정보 변경 (주소, 전화, 계좌) 반영

**수시 업무**
- 위촉 자격 미충족자 고지 및 상담
- 중복 위촉 여부 확인
- 위촉 서류 미비자 독촉

**어드민 관련 필드**: (미구현) 설계사 위촉 현황 대시보드 필요

---

### 1.2 계약 보전(배서) 처리

**정기 업무**
- 담보 추가/제외 신청 접수 및 검토
- 결제 수단 변경 (카드→계좌, 계좌번호 변경) 처리
- 계약자/납입자 변경 신청 검토
- 배서 처리 결과 보험사 회신 확인
- 배서 완료 고객 통지

**체크포인트**
- 담보 추가 시 심사 완료 여부 확인 (재심사 필요 케이스)
- 결제 수단 변경 시 계약 유효성 확인
- 납입자 변경 시 보험료 귀속 명확화

**어드민 관련 필드**: (미구현) 배서 유형 및 상태 추적 필요

---

### 1.3 보험료 수납 관리

**정기 업무**
- 신계약 수납 현황 모니터링 (추심/미납)
- 계속분 수기결제 접수 및 처리
- 카드/계좌이체 수납 현황 확인
- 미납 예정 고객 사전 안내 (카카오톡/SMS)
- 연체 고객 추심 전 단계 진행

**수납 프로세스**
| 구분 | 담당 | 확인 사항 |
|------|------|----------|
| 신계약 수납 | 상담팀 | 청약 접수 → 수납완료 |
| 계속분 수기결제 | 총무팀 | 고객 신청 → 결제확인 → 보험사 반영 |
| 미납 안내 | 관리팀 | 예정일 5일 전 1차, 연체 후 2차 |
| 추심 대행 | 외주업체 | 30일 이상 연체 건 |

**어드민 관련 필드**: 수납 현황 대시보드, 미납 추이 그래프 필요

---

### 1.4 신청/접수 반영

**정기 업무**
- 접수된 청약서 어드민 입력 (수동 데이터 입력)
- 고객 신청서 정보 검증 (중복 확인, 이상 값 검수)
- 보험사 청약 시스템 반영 (EDI 전송 또는 수동)
- 청약 반영 상태 추적 (반영 대기 → 완료 → 오류)

**수시 업무**
- 청약 반영 오류 수정 (고객 정보 정정 신청)
- 청약서 미제출 건 독촉
- 종이 청약서 원본 관리 (스캔, 보관)

**어드민 관련 필드**: 청약 현황 탭, 반영 상태 필드 (대기/성공/오류)

---

### 1.5 해촉자 계약 이관

**프로세스**
1. 설계사 해촉 신청 접수
2. 해촉자 보유 계약 목록 추출
3. 인수 설계사 배정 (본사/타사 협력사)
4. 계약자 변경 신청 접수 및 처리
5. 이관 완료 확인 (보험사 반영)
6. 해촉자 및 인수자 통지

**특수 케이스**
- 해촉자의 보험료 선불금 정산
- 미수금 회수 절차
- 계약 인수 거부 시 재배정

**어드민 관련 필드**: 해촉 현황 관리, 이관 추적 필요

---

### 1.6 종이 청약서 관리 (오프라인)

**업무 프로세스**
- 팩스/우편 수신 청약서 수령
- OCR 스캔 처리 (또는 수동 입력)
- 원본 서류 분류 및 보관 (설계사별/월별)
- 폐기 기준 적용 (5년 이상 경과 건)

**문서 보관 기준**
- 보관 장소: 금고 또는 전용 문서 캐비닛
- 접근 권한: 총무팀장 + 관리팀장만
- 폐기 승인: CEO 결재

---

### 1.7 개인정보 문서 징구/보관

**수집 대상**
- 신분증 사본 (설계사)
- 통장 사본 (수수료 입금 계좌)
- 납세 증명서 (연간 정산 시)
- 위탁 동의서 (청구 대행)
- 개인정보 수집 동의서 (GDPR/PIPA)

**수집 시점**
| 단계 | 수집 정보 | 담당팀 |
|------|---------|--------|
| 위촉 단계 (S1) | 신분증, 통장 | 총무팀 |
| 신청 단계 (S3) | 고객 신분증, 진료이력 | 상담팀 |
| 미팅 단계 (S7) | 계좌 정보 (지급 계좌) | 영업팀 |
| 청구 단계 (S10) | 위탁동의서 | 청구팀 |

**보관 규칙**
- 별도 잠금 장치가 있는 보관함 사용
- 전자 저장 시 암호화 처리
- 보유 기한: 거래 종료 후 5년
- 폐기 시 세단 처리 또는 소각

---

### 1.8 민원/완전판매/유지율 관리

**정기 업무**
- VOC 현황 모니터링 (수신사별 민원 접수)
- 민원 분류 (상품 관련/서비스 관련/기타)
- 완전판매 조사 응답 (금감원/보험협회)
- 유지율 추이 관리 (월별/설계사별)

**원수사 대응**
- 보험사 민원 회신 기한 (5일 이내)
- 민원 사유 분석 및 개선 건의
- 재민원 방지 대책 수립
- 분쟁해결 신청 처리

**어드민 관련 필드**: VOC 대시보드, 민원 현황 리스트, 처리 상태 추적

## 2. 준법감시팀 업무 (Compliance)

### 2.1 단계별 법률검토 추적 (S1~S17, Q1~Q9, R1~R14)

**검토 항목**
- 마케팅 문구 (광고법 준수)
- 청약서 양식 (금감원 규제)
- 계약서 내용 (약관 준수)
- 수수료 약정 (투명성 규제)
- 고객 보호 규정 (소비자보호법)

**체크포인트별 검토**

| Phase | 검토 항목 | 관여 시점 | 담당 |
|-------|---------|---------|------|
| S1. 유입 | 광고 마테리얼 | 광고 게시 전 | 준법감시 |
| S2-S3. 조회/신청 | 앱 가이드 텍스트 | 앱 업데이트 시 | 준법감시 |
| S4. 배정 | 배정 DB 조건 합법성 | 배정 규칙 수립 시 | 준법감시 |
| S5-S6. TM | 스크립트 및 녹취 | 매주 QA | 준법감시 + 콜팀장 |
| S7-S9. 미팅/계약 | 청약서/계약서 | 계약 전 최종 검토 | 준법감시 |
| S10-S13. 청구/분석 | 청구 대행약정서 | 청구 시작 전 | 준법감시 |
| S14-S15. 지급/사후 | 수수료 적정성 | 지급 결정 시 | 준법감시 |
| S16-S17. Growth | 소개 인센티브 | 인센티브 공고 전 | 준법감시 |

**Q(간편청구) 검토**
- Q1-Q2: 간편청구 신청 양식 검토
- Q3-Q4: 심사 기준의 공정성
- Q5-Q6: 보험사 청구 조건 준수
- Q7: 수수료 명시 및 투명성
- Q8-Q9: 사후 관리 프로세스

**어드민 관련**: 검토 체크리스트 자동 생성, 검토 상태 추적 필요

---

### 2.2 마케팅 소재 준법 심의

**심의 프로세스**

1. **소재 제출** (마케팅팀 → 준법감시)
   - 광고 이미지/영상/문안
   - 메타/구글/카카오 각 채널별 소재
   - SNS/인플루언서 협력 콘텐츠

2. **검토 항목**
   - 허위 표현 (보장 범위, 환급금 과장 등)
   - 오인 유발 (상품 비교 왜곡, 타사 폄하 등)
   - 의료 광고 기준 위반 (의료용어 무단 사용)
   - 개인정보 노출 위험

3. **검토 결과**
   - 승인 (법률상 문제 없음)
   - 조건부 승인 (수정 후 재제출)
   - 반려 (법률 위반, 재작성 필요)

4. **보관 및 추적**
   - 승인된 소재 버전 관리
   - 게시 기간 및 노출 현황 기록
   - 민원 발생 시 소재 추적 가능

**월별 심의 현황 보고**: CEO/CFO 보고용 리포트

**어드민 관련**: 마케팅 심의 대시보드, 소재 버전 관리, 승인 현황 통계

### 2.3 녹취 QA / 해피콜 모니터링

**녹취 QA**

**주간 모니터링**
- 전주 TM 녹취 파일 수집 (1차, 2차)
- 무작위 샘플링 (주당 20-30건, 콜팀별 5-10건)
- 스크립트 준수 여부 확인
- 부정 판매 행위 적발 여부 검토

**체크리스트**
| 항목 | 확인 내용 | 미충족 시 |
|------|---------|----------|
| 상품 설명 | 보장 범위, 보험료, 제외사항 |  재교육 |
| 수수료 안내 | 10% 명확 안내 | 경고 |
| 개인정보 | 수집 동의 취득 여부 | 재녹취 |
| 통화 품질 | 고객 이탈/거절 사항 | 태도 개선 지도 |
| 법령 위반 | 강압/오인/허위 표현 | 징계 건의 |

**적발 기준**
- 1회: 구두 경고 + 재교육
- 2회: 형식적 경고장
- 3회 이상: 징계 건의 (감봉/해고)

**해피콜 모니터링**
- 계약 완료 후 5-7일 후 고객 접촉
- 상품 설명 명확성 재확인
- 불만 고객 조기 발견
- 재계약 유도

**어드민 관련**: 녹취 첨부 확인, QA 체크리스트 기록, 부적격 내역 추적

---

### 2.4 민원 예방/대응 및 징계 건의

**민원 예방**
- 월 1회 준법 교육 시 주요 민원 사례 공유
- 완전판매 조사 응답 통계 분석
- 주요 민원 주제 연간 리포트

**민원 대응**
1. 접수 (금감원, 보험협회, 소비자원 등)
2. 사실 관계 조사 (설계사/콜팀 경청)
3. 원인 분석 (상품 설명 미흡/오류 등)
4. 대응안 수립 (사과, 환급, 배상 등)
5. 회신 (기한: 5일 이내)
6. 사후 개선 대책

**징계 건의**
- 대상: 정기 위반(3회 이상), 심각한 법령 위반, 재민원 발생
- 절차: 사실 확인 → 본인 반성 기회 → 징계 위원회 상정
- 수위: 구두 경고 → 경고장 → 감봉 → 해고

**어드민 관련**: VOC 등록/추적, 처리 이력 기록, 징계 현황 대시보드

---

### 2.5 준법 교육 운영 (월 1회)

**교육 대상**: 설계사/콜팀/영업팀/관리팀 전체

**월별 주제**

| 월 | 주제 | 대상 |
|-----|------|------|
| 1월 | 연간 규제 변화 및 정책 | 전체 |
| 2월 | 상품 설명 및 고지 의무 | 영업팀 |
| 3월 | 개인정보보호 및 정보보안 | 전체 |
| 4월 | 광고 및 마케팅 기준 | 마케팅팀 |
| 5월 | 불공정 거래행위 | 영업팀 |
| 6월 | 녹취 분석 및 사례 | 콜팀 |
| 7월 | 특별이익 제공 금지 | 영업팀 |
| 8월 | 소비자 민원 대응 | 전체 |
| 9월 | 중간 평가 및 피드백 | 전체 |
| 10월 | 고객 보호 기준 | 전체 |
| 11월 | 연말 결산 및 검점 | 전체 |
| 12월 | 내년도 계획 및 의견 수렴 | 전체 |

**운영 방식**
- 온라인/오프라인 혼합 (줌 + 회의실)
- 교육 시간: 1시간 (이론 40분 + Q&A 20분)
- 참석 확인: 출석부 작성, 부재자 별도 동영상 학습 지정
- 평가: 월말 스몸볼(3문제 객관식)

**어드민 관련**: 교육 일정 공고, 참석 현황 기록, 교육 자료 보관

### 2.6 위탁업체 점검 (월 1회 현장감사)

**감시 대상 업체**
- 외주 콜센터 (1차 TM 위탁업체 등)
- 청구 대행사 (보험사 청구 제출)
- OCR 스캔 업체
- 우편/택배 대행 업체

**감사 항목**

| 항목 | 확인 내용 | 주기 |
|------|---------|------|
| 준법 준수 | 스크립트, 녹취 관리 | 월 1회 |
| 개인정보보호 | 보관 시설, 접근 통제 | 월 1회 |
| 품질 관리 | 업무 오류율, 민원 현황 | 월 1회 |
| 문서 보관 | 서류 보안, 폐기 절차 | 분기 1회 |
| 계약 준수 | SLA 달성도, 비용 정산 | 월 1회 |

**감사 결과**
- 우수: 계약금 인센티브 (+2%)
- 일반: 계약금 유지
- 미흡: 개선 통보 (2주 내 시정)
- 심각: 계약 해지 검토

**어드민 관련**: 감사 결과 기록, 개선 현황 추적

### 2.7 Chinese Wall / 정보분리 모니터링

**정의**: 상충 가능성이 있는 부서 간 정보 차단

**적용 사례**

| 차단 대상 | 이유 | 조치 |
|----------|------|------|
| 콜팀 ↔ 영업팀 | 배정 DB와 상담 내용 분리 | 메뉴 접근 제한 |
| 영업팀 ↔ 청구팀 | 계약 권유 이익 충돌 | 청구 과정 투명성 보장 |
| 준법감시 ↔ 영업 | 감시 독립성 유지 | 별도 조직 운영 |
| 관리팀 ↔ 설계사 | 위촉/해촉 공정성 | 기준 공개 및 문서화 |

**모니터링 방식**
- 주간 접근 로그 리뷰 (정보 유출 여부)
- 월간 시스템 권한 감사 (부정 접근 적발)
- 분기 교육 (정보 분리 원칙 재강조)

**적발 시 조치**
- 경고: 1회 위반
- 징계: 2회 이상 또는 의도적 위반

**어드민 관련**: 사용자 권한 관리, 접근 로그 기록

### 2.9 특별이익 제공 모니터링

**특별이익의 정의**
- 계약 유도를 위한 금품 제공
- 과도한 선물/음식 접대
- 불공정한 수수료 할증
- 가이드라인 외 인센티브

**금지 행위**

| 행위 | 기준 | 적발 시 조치 |
|------|------|-----------|
| 금품 제공 | 1건당 10만원 초과 | 징계 + 환수 |
| 접대 | 1회당 5만원 초과 | 경고 |
| 수수료 조정 | 공시 기준 ±5% 초과 | 정정 + 환수 |
| 무보험 설계 | 계약자 동의 없는 보험 | 취소 + 징계 |

**모니터링 방식**
- 월 1회 경비 영수증 샘플 감시
- 분기 1회 설계사 인터뷰
- 고객 민원 접수 시 즉시 조사

**어드민 관련**: 특별이익 신고 폼, 감시 기록 보관

### 2.10 표준 스크립트 준법 검토 승인

**스크립트 관리**

**1차 TM 스크립트**
- 인사 → 신원 확인 → 상품 설명 → 수수료 안내 → 미팅 예약
- 주요 체크: 보장 설명 정확성, 수수료 10% 명시, 강압 표현 금지

**2차 TM 스크립트**
- 미팅 확정 → 준비물 안내 → 기존 설계사 관계 확인
- 주요 체크: 지연 이유 추궁 금지, 개인정보 재수집 금지

**미팅/계약 스크립트**
- 보장분석 → 계약서 설명 → 청약 진행
- 주요 체크: 약관 설명 완료, 청약 동의 명확화, 철회 권리 고지

**수정/업데이트**
1. 준법감시팀이 연 4회 이상 검토 시점 지정
2. 이해관계자 의견 수렴 (콜팀장, 영업팀장)
3. 수정안 준법 검토
4. CEO 승인 후 전사 공고
5. 교육 및 확인서 서명

**어드민 관련**: 스크립트 버전 관리, 검토 이력 기록

### 2.11 개인정보 수집/보관 모니터링

**수집 흐름 (여정별)**

| Phase | 수집 항목 | 수집처 | 보관 기간 |
|-------|----------|--------|---------|
| **S1 유입** | 광고 소스 (UTM) | 앱 자동 | 계약 종료 후 5년 |
| **S2 조회** | 생년월일, 휴대폰 번호 | 고객 입력 (앱) | 인증 후 즉시 삭제 또는 암호화 |
| **S3 신청** | 신분증 번호, 진료이력 | 고객 제출 + 심평원 API | 계약 종료 후 5년 |
| **S5 상담** | 통화 녹취 | 자동 녹음 | 6개월 (준법 검토 후) |
| **S7 미팅** | 계약자 정보, 계좌 정보 | 계약서 | 계약 종료 후 5년 |
| **S10 청구** | 청구 관련 서류 | 고객 제출 또는 병원 조회 | 청구 완료 후 3년 |

**보관 시설 기준**
- 물리적 보안: 잠금 문서함 (금고)
- 전자적 보안: 암호화 + 접근 제어
- 접근 인원: 필요 최소한 (총무팀장, 관리팀장, 준법감시팀장)

**폐기 절차**
1. 보유 기한 도래 확인
2. 폐기 대상 리스트 작성
3. CEO 승인
4. 세단 처리 (종이) 또는 완전 삭제 (전자)
5. 폐기 확인서 서명

**감시 항목**

| 항목 | 확인 내용 | 점검 주기 |
|------|---------|---------|
| 동의 취득 | 개인정보 수집 동의서 보관 여부 | 월 1회 샘플 감시 |
| 접근 통제 | 불필요한 접근 여부 | 월 1회 로그 검토 |
| 암호화 | 저장 데이터 암호화 상태 | 분기 1회 |
| 폐기 | 폐기 절차 준수 여부 | 분기 1회 |

**어드민 관련**: 개인정보 수집 현황 대시보드, 동의서 관리, 폐기 이력 기록

### 체크포인트별 상세 내용

**[CP1] 광고/마케팅 심의 (Phase 1)**
- 타이밍: 광고 게시 전 (최소 2일 전)
- 검토 항목: 허위/과장/오인 표현, 의료용어 사용, 개인정보 노출
- 결과: 승인/조건부 승인/반려
- 보관: 승인 소재 버전 + 게시 현황 기록

**[CP2] 앱 정보 검증 (Phase 2)**
- 타이밍: 앱 업데이트 시 또는 연 4회 정기 검토
- 검토 항목: 상품 설명 정확성, 수수료 명시, 보장 범위 표현
- 결과: 수정 필요/승인
- 문서화: 검토 의견 및 수정 이력 기록

**[CP3] 배정 기준 합법성 (Phase 3)**
- 타이밍: 배정 정책 수립 또는 변경 시
- 검토 항목: 나이/성별 차별 여부, 투명한 기준인지, 거부 사유 명확성
- 결과: 기준 승인 또는 수정 요구
- 주기: 월 1회 배정 현황 모니터링

**[CP4] 스크립트 및 녹취 QA (Phase 4)**
- 타이밍: 주 1회 (월요일/목요일)
- 검토 대상: 전주 TM 녹취 무작위 샘플(콜팀별 5-10건)
- 검토 항목: 상품 설명 정확성, 수수료 안내, 강압 표현 금지, 개인정보 보호
- 결과: 적격/부적격 판정
- 부적격 기준: 3회 연속 부적격 시 징계 건의
- 보관: QA 체크리스트 + 부적격 녹취 보관

**[CP5] 청약서/계약서 최종 검토 (Phase 5)**
- 타이밍: 계약 실행 전 (당일 또는 전일)
- 검토 항목: 상품 설명 완료 기재, 청약 동의 서명, 철회 권리 안내 확인
- 대상 샘플: 일 3-5건 랜덤 추출
- 결과: 적격(진행 가능)/부적격(재확인 필요)
- 부적격 조치: 영업팀 재교육

**[CP6] 청구 조건 및 서류 검증 (Phase 6)**
- 타이밍: 청구 신청 접수 시
- 검토 항목: 청구 적격성(보장 조건), 서류 완결성, 청구대행약정서 확인
- 결과: 승인(진행)/보류(추가 서류 필요)/거절(부적격)
- 보관: 청구 검토 내역 + 거절 사유 기록

**[CP7] 수수료 적정성 검증 (Phase 7)**
- 타이밍: 지급 예정 7일 전
- 검토 항목: 수수료율 규정 준수, 부정 조정 여부, 특별이익 제공 여부
- 표본: 월 20-30건 무작위 추출
- 결과: 정상/의심(조사)/오류(정정)

**[CP8] 소개 인센티브 심의 (Phase 8)**
- 타이밍: 인센티브 정책 수립/변경 시
- 검토 항목: 금액 적정성, 법적 규제 준수, 특별이익 해당 여부
- 결과: 승인/조건부 승인/거절
- 문서화: 승인 공고 + 실행 이력
