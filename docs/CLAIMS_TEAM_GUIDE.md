# 📋 청구팀 — 담당 구간 업무 상세 가이드

> 담당 Phase: 청구/분석(Phase 6) + 지급/사후(Phase 7)

> 팀 구성: 청구팀원(분석·접수·서류) + 청구팀장(배정·외근·리뷰)

```

### Q3. 1차 청구콜 / 서류 확보

**담당**: 청구팀원

| 업무 | 구분 | 설명 | 어드민 구현 |
|------|------|------|---------|
| 고객 전화 발신 (1차 청구콜) | [OFF] | 간편청구 신청 고객에게 전화 (콜스크립트 활용) | 오프라인 (전화) |
| **콜 스크립트 체크리스트** | [ON] | 항목별 체크: 보험확인/서류확보/진행동의 | ❌ Q 전용 스크립트 없음 |
| 서류 확보 상태 관리 | [ON] | 신청서/건보료/병원기록 등 필수 서류 수집 | ❌ 워크플로우 없음 |
| **콜 연결률 / 서류 확보율 KPI** | [ON] | 관리자 모니터링 | ❌ Q KPI 0개 |

#### 콜 스크립트 주요 내용

```
1. 신원확인: 고객명/생년월일/계약번호 확인
2. 보험 현황: 현재 가입 보험료 및 납입 상태 확인
3. 청구 범위: 최근 3년 치료 이력 설명
4. 필요 서류:
   - 건강보험료 고지서 (최근 1개월)
   - 병원/약국 방문 기록 사본
   - 신분증 사본
5. 동의: 서류 발송 및 청구 진행 동의 획득
6. 예상 환급액 안내 (심평원 기반 추정)
```

### Q4. 1차 정밀 분석 (교차대조) — 간편청구 핵심

|------|------|------|---------|
| 심평원 / 홈텍스 / 건보 데이터 교차 조회 | [EXT] | 3개 기관 데이터 수집 (심평원/국세청/건보) | ❌ 교차대조 연동 없음 |
| 데이터 교차 대조 분석 화면 | [ON] | 3개 데이터 비교 분석 → 미청구/중복청구 발굴 | ❌ 미구현 |
| 미청구건 발굴 | [ON] | "왜 이 치료는 청구가 안 되었을까?" 발굴 | ❌ 미구현 |
| 청구판단서 자동 생성 | [ON] | 분석 결과 기반 "어떤 건을 청구할 것인가" 정리 | ❌ 미구현 |

#### 교차대조 분석 예시

```
[심평원 데이터]        [국세청 홈텍스]        [건보료]

뇌졸중 진료 2023.08 보험료 공제 미기재     연 102만
                                        → 보험 최소요건 O

분석결과:
✓ 암 진료 청구 가능
✓ 뇌졸중 진료 청구 가능
✓ 치료비 영수증 필요
→ 추정 환급액: 200만원
```

│ 소요시간: 2~3주 | 외근팀 활용: 거의 없음 | 복잡도: 낮음         │
│ 핵심: 교차대조 분석 → 미청구건 발굴 → 빠른 청구                  │
└─────────────────────────────────────────────────────────────────┘
```

## 👥 청구팀원 vs 청구팀장 역할 분담

### 청구팀원 (claims_member)

**주요 책임**:
- S10: 청구 접수 (신분증 체크, 서류 수집)
- S11: 미지급금 분석
- S12: 발급 필요 서류 생성, OCR 처리, ENFAX 팩스
- S14: 지급 결과 확인, 고객 정산
- Q3: 1차 청구콜, 서류 확보
- Q4~Q7: 분석 → 청구 → 지급추적

**특징**: 각 고객별 1:1 담당 → 깊이 있는 분석 · 꼼꼼한 서류 관리

---

### 청구팀장 (claims_lead)

**주요 책임**:
- S10: 청구DB 배정 (오전 일일 루틴: 엑셀 정리 → 팀원 배정)
- S12: 외주업체(투에이치) 발급 요청 관리, 외근팀 상태 추적
- S13: 최종 분석 리뷰 (OCR 결과 + 수집 서류 검증)
- S12~S13: 발급 진행 상태 모니터링 (팀원별 진행률)

**특징**: 프로세스 오케스트레이션 · 외부기관 관리 · 품질 리뷰
