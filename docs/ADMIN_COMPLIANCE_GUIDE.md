# ⚙️ 관리/지원팀 — 전 여정 횡단 업무

> **역할**: 직접 고객 대응은 없으나 전 여정에서 운영 지원 및 준법 감독
> **핵심 키워드**: 위촉관리 / 계약보전 / 준법감시 / 개인정보보호
> **팀 구성**:
> - **관리팀** (admin): 위촉관리, 계약보전, 수납, 정산, 개인정보
> - **준법감시팀** (compliance): 법률검토, 마케팅심의, 녹취QA, 민원대응, 교육

---

## 1️⃣ 총무/인사팀 업무

### 1.1 위촉/해촉 관리

**정기 업무**
- 신규 설계사 협회 등록 절차 지원 (보험대리점협회, 손보협회)
- 위촉장 발급 및 보관 관리
- 설계사 정보 변경사항 반영 (주소, 전화, 계좌)
- 해촉 요청 접수 및 처리

**수시 업무**
- 위촉 자격 미충족자 상담
- 중복 위촉 여부 확인
- 위촉 서류 미비자 독촉

**어드민**: (미구현) 설계사 위촉 현황 대시보드 필요

---

### 1.2 계약 보전(배서) 처리

**정기 업무**
- 담보 추가/제외 신청 접수
- 결제 수단 변경 (카드→계좌, 계좌번호 변경) 처리
- 계약자/납입자 변경 신청 검토
- 배서 처리 결과 보험사 회신 확인
- 배서 완료 고객 통지

**체크포인트**
- 담보 추가 시 심사 완료 여부 확인 (재심사 필요 케이스)
- 결제 수단 변경 시 계약 유효성 확인
- 납입자 변경 시 보험료 귀속 명확화

**어드민**: (미구현) 배서 유형 및 상태 추적 필요

---

### 1.3 보험료 수납 관리

**정기 업무**
- 신계약 수납 현황 모니터링 (추심/미납)
- 계속분 수기결제 접수 및 처리
- 카드/계좌이체 수납 현황 확인
- 미납 예정 고객 사전 안내 (카카오톡/SMS)
- 연체 고객 추심 전 단계 진행

**수납 프로세스**

| 구분 | 담당 | 확인 사항 |
|------|------|----------|
| 신계약 수납 | 상담팀 | 청약 접수 → 수납완료 |
| 계속분 수기결제 | 총무팀 | 고객 신청 → 결제확인 → 보험사 반영 |
| 미납 안내 | 관리팀 | 예정일 5일 전 1차, 연체 후 2차 |
| 추심 대행 | 외주업체 | 30일 이상 연체 건 |

**어드민**: 수납 현황 대시보드, 미납 추이 그래프 필요

---

### 1.4 신청/접수 반영

**정기 업무**
- 접수된 청약서 어드민 입력 (수동 데이터 입력)
- 고객 신청서 정보 검증 (중복 확인, 이상 값 검수)
- 보험사 청약 시스템 반영 (EDI 전송 또는 수동)
- 청약 반영 상태 추적 (반영 대기 → 완료 → 오류)

**수시 업무**
- 청약 반영 오류 수정 (고객 정보 정정 신청)
- 청약서 미제출 건 독촉
- 종이 청약서 원본 관리 (스캔, 보관)

**어드민**: 청약 현황 탭, 반영 상태 필드 (대기/성공/오류)

---

### 1.5 해촉자 계약 이관

**프로세스**
1. 설계사 해촉 신청 접수
2. 해촉자 보유 계약 목록 추출
3. 인수 설계사 배정 (본사/타사 협력사)
4. 계약자 변경 신청 접수 및 처리
5. 이관 완료 확인 (보험사 반영)
6. 해촉자 및 인수자 통지

**특수 케이스**
- 해촉자의 보험료 선불금 정산
- 미수금 회수 절차
- 계약 인수 거부 시 재배정

**어드민**: 해촉 현황 관리, 이관 추적 필요

---

### 1.6 종이 청약서 관리 (오프라인)

**업무 프로세스**
- 팩스/우편 수신 청약서 수령
- OCR 스캔 처리 (또는 수동 입력)
- 원본 서류 분류 및 보관 (설계사별/월별)
- 폐기 기준 적용 (5년 이상 경과 건)

**문서 보관 기준**
- 보관 장소: 금고 또는 전용 문서 캐비닛
- 접근 권한: 총무팀장 + 관리팀장만
- 폐기 승인: CEO 결재

### 1.7 개인정보 문서 징구/보관

**수집 대상**
- 신분증 사본 (설계사)
- 통장 사본 (수수료 입금 계좌)
- 납세 증명서 (연간 정산 시)
- 위탁 동의서 (청구 대행)
- 개인정보 수집 동의서 (GDPR/PIPA)

**수집 시점**

| 단계 | 수집 정보 | 담당팀 |
|------|---------|--------|
| 위촉 단계 (S1) | 신분증, 통장 | 총무팀 |
| 신청 단계 (S3) | 고객 신분증, 진료이력 | 상담팀 |
| 미팅 단계 (S7) | 계좌 정보 (지급 계좌) | 영업팀 |
| 청구 단계 (S10) | 위탁동의서 | 청구팀 |

**보관 규칙**
- 별도 잠금 장치가 있는 보관함 사용
- 전자 저장 시 암호화 처리
- 보유 기한: 거래 종료 후 5년
- 폐기 시 세단 처리 또는 소각

### 1.8 민원/완전판매/유지율 관리

**정기 업무**
- VOC 현황 모니터링 (수신사별 민원 접수)
- 민원 분류 (상품 관련/서비스 관련/기타)
- 완전판매 조사 응답 (금감원/보험협회)

**원수사 대응**

- 민원 사유 분석 및 개선 건의
- 재민원 방지 대책 수립
- 분쟁해결 신청 처리

**어드민**: VOC 대시보드, 민원 현황 리스트, 처리 상태 추적

## 2️⃣ 준법감시팀 업무

### 2.1 여정별 법률검토 추적

여정의 각 단계에서 준법감시가 개입하여 법률 검토 및 규제 준수를 확인합니다.

**단계별 검토 기준**

| Phase | 여정 스텝 | 검토 항목 | 관여 시점 | 담당 |
|-------|---------|---------|---------|------|
| 유입 | S1, R1, Q1 | 광고 마테리얼 | 광고 게시 전 | 준법감시 |
| 조회/신청 | S2-S3, R2-R3, Q2 | 앱 가이드 텍스트 | 앱 업데이트 시 | 준법감시 |
| 선별/배정 | S4 | 배정 DB 조건 합법성 | 배정 규칙 수립 시 | 준법감시 |
| 상담/TM | S5-S6, R4, Q2 | 스크립트 및 녹취 | 매주 QA | 준법감시 + 담당팀장 |
| 미팅/계약 | S7-S9, R5-R6 | 청약서/계약서 | 계약 전 최종 검토 | 준법감시 |
| 청구/분석 | S10-S13, R7-R10, Q3-Q6 | 청구 대행약정서 | 청구 시작 전 | 준법감시 |
| 지급/사후 | S14-S15, R11-R12, Q7 | 수수료 적정성 | 지급 결정 시 | 준법감시 |
| Growth | S16-S17, R13-R14, Q8-Q9 | 소개 인센티브 | 인센티브 공고 전 | 준법감시 |

**검토 항목 (공통)**
- 마케팅 문구 (광고법 준수)
- 청약서 양식 (금감원 규제)

- 수수료 약정 (투명성 규제)
- 고객 보호 규정 (소비자보호법)

**어드민**: 검토 체크리스트 자동 생성, 검토 상태 추적 필요

### 2.2 마케팅 소재 준법 심의

**심의 프로세스**

1. **소재 제출** (마케팅팀 → 준법감시)
   - 광고 이미지/영상/문안
   - 메타/구글/카카오 각 채널별 소재
   - SNS/인플루언서 협력 콘텐츠

2. **검토 항목**
   - 허위 표현 (보장 범위, 환급금 과장 등)
   - 오인 유발 (상품 비교 왜곡, 타사 폄하 등)
   - 의료 광고 기준 위반 (의료용어 무단 사용)
   - 개인정보 노출 위험

3. **검토 결과**
   - 승인 (법률상 문제 없음)
   - 조건부 승인 (수정 후 재제출)
   - 반려 (법률 위반, 재작성 필요)

4. **보관 및 추적**
   - 승인된 소재 버전 관리
   - 게시 기간 및 노출 현황 기록
   - 민원 발생 시 소재 추적 가능

**월별 심의 현황 보고**: CEO/CFO 보고용 리포트

**어드민**: 마케팅 심의 대시보드, 소재 버전 관리, 승인 현황 통계

### 2.3 녹취 QA / 해피콜 모니터링

**녹취 QA (주간)**
- 전주 TM 녹취 파일 수집 (1차, 2차)
- 무작위 샘플링 (주당 20-30건, 콜팀별 5-10건)
- 스크립트 준수 여부 확인
- 부정 판매 행위 적발 여부 검토

**QA 체크리스트**

| 항목 | 확인 내용 | 미충족 시 |
|------|---------|----------|
| 상품 설명 | 보장 범위, 보험료, 제외사항 | 재교육 |
| 수수료 안내 | 10% 명확 안내 | 경고 |
| 개인정보 | 수집 동의 취득 여부 | 재녹취 |
| 통화 품질 | 고객 이탈/거절 사항 | 태도 개선 지도 |
| 법령 위반 | 강압/오인/허위 표현 | 징계 건의 |

**적발 기준**
- 1회: 구두 경고 + 재교육
- 2회: 형식적 경고장
- 3회 이상: 징계 건의 (감봉/해고)

**해피콜 모니터링**
- 계약 완료 후 5-7일 후 고객 접촉
- 상품 설명 명확성 재확인
- 불만 고객 조기 발견
- 재계약 유도

**어드민**: 녹취 첨부 확인, QA 체크리스트 기록, 부적격 내역 추적

### 2.4 민원 예방/대응 및 징계 건의

**민원 예방**
- 월 1회 준법 교육 시 주요 민원 사례 공유
- 완전판매 조사 응답 통계 분석
- 주요 민원 주제 연간 리포트

**민원 대응 프로세스**
1. 접수 (금감원, 보험협회, 소비자원 등)
2. 사실 관계 조사 (설계사/콜팀 경청)
3. 원인 분석 (상품 설명 미흡/오류 등)
4. 대응안 수립 (사과, 환급, 배상 등)
5. 회신 (기한: 5일 이내)
6. 사후 개선 대책

**징계 건의**
- 대상: 정기 위반(3회 이상), 심각한 법령 위반, 재민원 발생
- 절차: 사실 확인 → 본인 반성 기회 → 징계 위원회 상정
- 수위: 구두 경고 → 경고장 → 감봉 → 해고

**어드민**: VOC 등록/추적, 처리 이력 기록, 징계 현황 대시보드

### 2.7 Chinese Wall / 정보분리 모니터링

**정의**: 상충 가능성이 있는 부서 간 정보 차단

**적용 사례**

| 차단 대상 | 이유 | 조치 |
|----------|------|------|
| 콜팀 ↔ 영업팀 | 배정 DB와 상담 내용 분리 | 메뉴 접근 제한 |
| 영업팀 ↔ 청구팀 | 계약 권유 이익 충돌 | 청구 과정 투명성 보장 |
| 준법감시 ↔ 영업 | 감시 독립성 유지 | 별도 조직 운영 |
| 관리팀 ↔ 설계사 | 위촉/해촉 공정성 | 기준 공개 및 문서화 |

**모니터링 방식**
- 주간 접근 로그 리뷰 (정보 유출 여부)
- 월간 시스템 권한 감사 (부정 접근 적발)
- 분기 교육 (정보 분리 원칙 재강조)

**적발 시 조치**
- 경고: 1회 위반
- 징계: 2회 이상 또는 의도적 위반

**어드민**: 사용자 권한 관리, 접근 로그 기록

**어드민**: 기준 문서 보관/버전 관리, 준수 체크리스트 시스템화

### 2.9 특별이익 제공 모니터링

**특별이익의 정의**
- 계약 유도를 위한 금품 제공
- 과도한 선물/음식 접대
- 불공정한 수수료 할증
- 가이드라인 외 인센티브

**금지 행위 및 기준**

| 행위 | 기준 | 적발 시 조치 |
|------|------|-----------|
| 금품 제공 | 1건당 10만원 초과 | 징계 + 환수 |
| 접대 | 1회당 5만원 초과 | 경고 |
| 수수료 조정 | 공시 기준 ±5% 초과 | 정정 + 환수 |
| 무보험 설계 | 계약자 동의 없는 보험 | 취소 + 징계 |

**모니터링 방법**
- 월 1회 설계사 수수료 현황 검토
- 분기 1회 인센티브 프로그램 감시
- 민원 접수 시 즉시 조사

**어드민**: 특별이익 거래 기록, 모니터링 결과 추적

### 2.10 표준 스크립트 준법 검토 승인

**적용 대상**
- 1차 TM 스크립트 (S5)
- 2차 TM 스크립트 (S6)
- 미팅 전 설명 스크립트 (S7)
- 간편청구 콜 스크립트 (Q2)
- 해피콜 스크립트 (전체 여정)

**검토 기준**
- 상품 설명의 정확성 (보장/제외 명확화)
- 수수료 안내 투명성 (10% 명시)
- 고객 동의 절차 준수 (개인정보 동의)
- 강압/오인 표현 제거
- 법령 위반 문구 제거

**승인 프로세스**
1. 영업팀/콜팀 → 스크립트 초안 제출
2. 준법감시 → 검토 및 수정 의견
3. 재작성 및 재검토 (필요 시 반복)
4. 최종 승인 → 시행
5. 월 1회 준수 현황 모니터링

**어드민**: 스크립트 버전 관리, 승인 이력 기록

- 수납 현황 (신계약, 계속분)
- 청약 반영 상태
- 배서 처리 현황
- 민원 현황
- 개인정보 수집 내역

**제한된 영역**
- 상담 내용 조회 (음성 녹취 등)
- 영업팀 성과급 계산 (영업팀만)
- 청구 대행 결과 (청구팀장만)

### 준법감시팀 (compliance)

**메뉴 접근 권한**
- 전체 여정 조회 (감시 목적)
- 음성 녹취 청취 (QA 목적)
- 마케팅 소재 검토 (심의 대시보드)
- 스크립트 승인/관리
- 민원 현황 및 대응 이력
- 개인정보 수집/보관 모니터링
- 교육 현황 및 성적 관리
- 위탁업체 감사 기록

**제한된 영역**
- 개인 영업 실적 (성과급 등) — 투명성 위해 조회 가능하나 활용 제한
- 고객 상담 내용 편집 (기록 목적만)
